## 搬瓦工香港机房是什么？先搞清楚你在看哪个

搬瓦工目前在香港有两个机房，别搞混。

一个是老机房 HKHK_1，走 PCCW 线路（电盈），很久没有大更新，丢包率在高峰期问题比较多，现在基本不推荐。

另一个就是今天要说的 HKHK_8，官方称为 Hong Kong CN2 GIA，或者叫 Ultra-HK，是搬瓦工目前香港线路最强的机房。物理位置在香港九龙，走的是 CTGNet CN2 GIA（AS4809）出口，三网回程都经过 CN2 骨干。这是搬瓦工亚太区线路配置最高的机房之一，价格也对应最贵。

注意一点：HKHK_8 是专属套餐，买 CN2 GIA-E 套餐之后**没有办法**把机房迁移过来。想用香港 CN2 GIA 机房，只有一条路——单独买 HK CN2 GIA 套餐。

## 搬瓦工香港机房延迟数据

这是大部分人真正想知道的。

直接说数字：广东方向延迟大概在 9ms 上下，这基本是物理极限，再近也近不了多少了——香港到广东就那么远。华东（上海、浙江一带）实测在 39ms 左右。北方用户会稍微高一些，但整体仍然远低于美国机房。

这个延迟水平是晚高峰测出来的结果，不是空载时候跑出来的数字。CN2 GIA 线路的抗拥堵能力比 CN2 GT 强得多，高峰期不容易被路由绕道，这也是它比普通香港线路贵这么多的核心原因。

丢包方面，联通表现最好，实测零丢包。电信也接近零。移动方向有一些节点超时——广州和北京都有，所以如果你主要用移动 SIM 卡上网，建议先自己 ping 一下测试 IP（`89.208.246.218`），再决定要不要买。

## 硬件不是瓶颈

HKHK_8 现在跑的是 AMD EPYC-Genoa 处理器，也就是 AMD 第四代 EPYC（Zen 4 架构），比之前搬瓦工香港用的 Intel Xeon 是明确的硬件升级。磁盘用 NVMe SSD，bench.sh 综合 I/O 平均在 927 MB/s，4K 随机读写 IOPS 接近 94k，跑建站、跑数据库都不会卡在磁盘这里。

带宽是 1Gbps。能在香港 CN2 GIA 线路上做到 1Gbps 大带宽的 VPS 服务商，搬瓦工是为数不多能做到的。阿里云、腾讯云这些大厂的香港线路，带宽通常只有几兆到 30M 左右，要选配到 1Gbps 那价格完全没法看。

## 说实话：这个价格，适合谁？

搬瓦工香港 CN2 GIA 的入门价是 $89.99/月。这不是便宜套餐，是搬瓦工所有在售方案里价格最高的那一档。

用下来觉得最值回票价的场景大概是这几种：

**在华南做业务的。** 广东用户对香港的延迟天然有优势，9ms 的往返延迟几乎和本地服务器没区别。如果应用对响应速度很敏感（比如实时交易、低延迟 API 调用），这个机房确实能解决问题。

**需要香港真实 IP 的。** HKHK_8 给的是香港九龙 IP，对于跨境电商合规、香港公司在线服务、需要香港归属地的场景，这个 IP 是完全合规的选项。

**高峰期稳定性要求高的建站用户。** CN2 GIA 在晚高峰的表现远强于 CN2 GT 和普通国际线路，用来跑面向国内用户的网站，体验差别很明显。

问题是什么？价格。

$89.99/月折合人民币大概 650 元左右，不是个小数字。对比之下，搬瓦工 CN2 GIA-E 套餐季付 $49.99，一个季度还不到香港套餐一个月的价格，速度比香港稍慢，但延迟差距在美国机房和香港机房之间，对华北用户来说差距其实没那么大。

所以我的看法：在广东、对延迟极度敏感、或者业务需要香港 IP 的，这个溢价花得值。北方用户、预算有限的，先看 CN2 GIA-E。
